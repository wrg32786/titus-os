# Bio-hacking Posture

> The principal does not **consume** Claude Code. The principal **tunes** Claude Code. The framework is an organism being performance-engineered, not a product being used.

Every existing piece of Titus already implies this posture. This note makes it explicit and names the categories of intervention the posture demands.

## The frame

Bio-hackers don't passively accept their physiology. They measure, tweak, restrict, supplement, sleep-protocol, environment-design, stack, log outcomes, and iterate. Their relationship to their own body is *engineering*, not *consumption*.

Apply that to AI tooling and you get Titus. The principal isn't a user of an AI product; they're an operator tuning an organism. Every intervention has a category. Every intervention compounds. Every intervention is measured, even if loosely.

## The categories of intervention

### Supplements
Adding capability the system didn't have. **Skills, hooks, doctrine notes.** Each one is a deliberate addition that makes the framework more capable in a specific dimension.

### Sleep optimization
Maintaining the framework between sessions so each new one starts strong. **`/close` writes the daily, the heat index decays unused notes, MEMORY.md gets pruned.** Like a bio-hacker treating sleep as recovery, the framework treats session-close as the recovery cycle.

### Stack tracking
Knowing what's running and what it costs. **Token tracker, session summaries, skill audit, decision log.** You can't tune what you don't measure.

### Restriction
What the framework explicitly refuses. **The legibility thesis. The negative space. The "what I am not building" list.** Bio-hackers know their no-fly foods; the framework knows its no-build directions.

### Environmental design
Setting up the surroundings so the right things happen automatically. **`install.sh`, the conversational `/setup`, the hook architecture.** The framework's environment configures itself toward intended use.

### Stacks
Layering interventions that compound. **The seven-layer architecture (kernel → doctrine → vault → daemons → hooks → skills → user config).** Each layer does one thing; the stack as a whole does what no single layer could.

### Protocols
Cadenced interventions. **Daily (`/open`, `/close`), weekly (quality-pulse, sanitize-and-publish), monthly (vault-health, skill-audit), quarterly (retro, red-team).** A bio-hacker's morning supplement stack is a protocol; so is a Friday `/close`.

## What this posture rules out

- **Treating the framework as static.** A static framework rots. The posture demands ongoing tuning.
- **Adding without measuring.** Every supplement should produce signal — usage, outcome, or noticeable improvement. If a skill is invoked twice in 90 days, it's a candidate for retirement.
- **Restricting without writing it down.** "I don't do X" is a private rule. "I don't do X because Y, and the trigger to reconsider is Z" is doctrine.
- **Optimizing only one category.** Adding skills (supplements) without adding measurements (stack tracking) produces a bloated framework with no feedback loop. The posture demands all categories stay populated.

## What this posture forces in

- **Explicit measurement.** Token costs, skill invocations, decision outcomes 30/60/90 days later, a Friday quality pulse.
- **Documented restrictions.** A "what I am not building" list with reasons and reconsider-triggers.
- **Cadenced check-ins.** Not just `/open` and `/close` but `/skill-audit`, `/retro`, `/system-check`, `/diagnose`.
- **Adaptive pressure.** `/red-team` against the framework's own weakest skill. `/break-on-purpose` to prove the boundaries hold.
- **Compounding learning.** Every "I'm reading X" moment becomes a structured `/learn` capture. Past evaluations don't get forgotten.

## The bet

The bet is that frameworks tuned this way produce better outcomes per unit of principal attention than frameworks built once and consumed. Compounding > cleverness. Tuning > consuming.

If the bet is right, an organism-shaped framework matches the rate at which AI itself is changing. If the framework is static, it gets older every week. If it tunes, it gets sharper.
