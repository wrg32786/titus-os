# Vault Examples

Populated example notes showing what good vault content looks like. None of these are real — they're illustrative and entirely fabricated.

The example "Daisy" is a fictional founder running a 4-person SaaS startup, used across all examples for consistency.

## Files

- `concepts/Pricing Strategy.md` — example domain knowledge note
- `memory/SESSION_LOG.md` — example rolling session log entry
- `memory/ACTIVE_PRIORITIES.md` — example tier 1/2/3 priority stack
- `memory/DECISION_LOG.md` — example append-only decision history
- `projects/Onboarding v2.md` — example active project note
- `people/Engineering Lead.md` — example team-member note
- `daily/2026-04-25.md` — example session note (one day)

## What to do with these

When you fork or clone Titus and run your first session, the AI will set up your real `vault/concepts/`, `vault/memory/`, `vault/projects/`, etc. through conversation. These examples are just the shape — the texture of what populated content looks like.

Feel free to delete this `examples/` directory once you've gotten the picture, or keep it as reference. Nothing in the framework reads from it.
