# Engineering Lead

> [!info]
> Lead engineer on the team. Co-founder. Owns the engineering side end-to-end — architecture, hiring, code reviews, ship discipline.

## Working style

- **Strong opinions on architecture, weak opinions on tools.** Will push back hard if we propose something that breaks system invariants. Open to whatever language/framework if the architecture is right.
- **Prefers async briefs over meetings.** Structured PR descriptions and design docs land well; ad-hoc chat for architecture decisions doesn't.
- **High autonomy.** Doesn't need check-ins. Does need clear scope and a deadline.
- **Will tell me when I'm wrong.** Welcome that. Don't talk past it.

## What they own

- All technical architecture decisions (Level 1 / Level 2 per [[12_authority_matrix]])

- Code review and PR sign-off
- Production incident response

## What they don't own

- Pricing, GTM, marketing copy
- Customer-facing decisions (handle joint with me)
- Anything legal, financial, fundraising
