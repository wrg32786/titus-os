## The 3-Gate Filter

Every opportunity passes through three gates before getting time:

### Gate 1 — Pattern Fit
Does this match a pattern I know works?
- Is this a system that compounds over time?
- Does it leverage existing infrastructure?
- Is there a clear path from effort to outcome?

If it doesn't match a known pattern, it goes to the idea queue — not the active work.

### Gate 2 — Priority Stack
Where does this fit in the current priority hierarchy?

| Tier | Description | Time Allocation |
|------|-------------|----------------|
| Tier 1 | Income restoration / revenue | 60% |
| Tier 2 | Asset building / equity | 30% |
| Tier 3 | Leverage expansion / optionality | 10% |

New work must justify its tier. Tier 3 work that tries to claim Tier 1 time gets killed.

### Gate 3 — Asymmetry Test
- If this works, is the outcome meaningfully large relative to effort?
- If this fails, is the downside bounded?
- Is the ratio of upside to downside at least 5:1?

## Leverage Types (Priority Order)

1. **Automation leverage** — Build it once, runs forever. Highest priority.
2. **Distribution leverage** — Reach many people with one action. Second priority.
3. **IP and brand leverage** — Assets that appreciate over time. Third priority.
4. **Capital leverage** — Money working instead of time. Requires capital.
5. **Labor leverage** — Other people's time. Most expensive, least scalable.

## Operating Modes

| Mode | Trigger | Bias |
|------|---------|------|
| **Stabilization** | Zero or declining revenue | Cost awareness, fast wins, simplification, revenue restoration |
| **Build** | Revenue stable, growth opportunity clear | Invest in systems, hire, scale what works |
| **Expansion** | Revenue growing, multiple options | Place bets, explore new markets, increase leverage |
| **Recovery** | Something broke | Stop the bleeding, diagnose root cause, restore stability |

The operating mode determines which tier gets priority and how aggressive to be.

## Anti-Drift Rules

1. **3 active bets maximum.** Every new bet means killing or pausing an existing one.
2. **Idea queue, not idea execution.** New ideas go to the queue. They earn active status by passing the 3-gate filter.
3. **Weekly priority review.** Every week, confirm the priority stack is still correct. Priorities drift faster than you think.
4. **Kill fast.** If something isn't working after a fair test, kill it. Sunk cost is not a reason to continue.
