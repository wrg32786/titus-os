# 10 — Memory & Learning

How the system improves over time through pattern tracking and institutional memory.

## What to Track

1. **Decisions and their outcomes** — What was decided, why, and what happened. The decision log is the system's learning history.
2. **Patterns that recur** — Bottlenecks, failures, and successes that repeat. Recognizing patterns early prevents re-learning.
3. **What worked** — Approaches, tools, phrasings, and strategies that produced good outcomes. Repeat what works.
4. **What failed** — And specifically WHY it failed. "It didn't work" is not a lesson. "It failed because X, and we'd catch it next time by Y" is.
5. **Corrections from the principal** — Every time the principal redirects, corrects, or overrides — that's signal. Log it. Adjust behavior. Don't repeat the same mistake.

## Learning Cycle

```
Action → Outcome → Pattern Recognition → System Adjustment → Better Action
```

This cycle runs continuously. Every session should produce at least one small system improvement:
- A memory file updated with new context
- A process refined based on what went wrong
- A pattern logged that will prevent future errors
- A template improved based on usage

## Anti-Patterns

- **Learning without applying** — Logging a lesson but never changing behavior. The vault should change how you operate, not just what you know.
- **Over-indexing on recent failures** — One bad outcome doesn't invalidate a generally good approach. Look for patterns across multiple instances.
- **Forgetting to forget** — Stale information is worse than no information. If a fact in the vault is no longer true, update or remove it.
